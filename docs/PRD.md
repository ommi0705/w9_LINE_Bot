# 股票投資小幫手 LINE Bot PRD

## 1. 執行摘要 (Executive Summary)
* **背景動機 (Background):** 現代投資人生活忙碌，無法時時刻刻盯盤或消化龐雜的財經新聞。LINE 是台灣最普及的通訊軟體，若能透過 LINE Bot 提供即時股價查詢、重大新聞推播，甚至結合 AI（如 Gemini）進行財經資訊摘要與名詞解釋，將能大幅降低投資資訊的取得門檻。
* **產品目標 (Objectives):** 打造一個便捷的 LINE 股票小幫手，提供快速的個股報價查詢、AI 新聞摘要以及到價提醒功能，提升使用者的投資決策效率與平台互動黏著度。
* **成功指標 (Success Metrics):** 
  * 上線首月獲得 100 位活躍使用者 (MAU)。
  * 每日平均訊息查詢次數 (Queries per day) 大於 500 次。
  * 到價提醒功能的設定轉換率 (設定人數 / 活躍人數) 達 20%。

## 2. 目標受眾 (Target Audience)
* **使用者畫像 (Persona 1 - 新手小資族):** 剛接觸股市，對專有名詞不熟，希望能有人話版的新聞解說與簡單的看盤方式。
* **使用者畫像 (Persona 2 - 忙碌上班族):** 有一定投資經驗，但上班無法隨時看盤，需要即時的到價提醒與收盤後的市場重點摘要。
* **使用場景 (Use Cases):** 
  * 通車通勤時，在 LINE 快速查詢手邊持股的最新股價。
  * 盤中設定「台積電跌破 1000 元」的通知，並在達到條件時立刻收到 LINE 訊息。
  * 遇到不懂的財經名詞或長篇新聞，傳送給 Bot 請 AI 幫忙濃縮重點。

## 3. 功能需求 (Functional Requirements)

* **功能 1: 即時個股報價查詢**
    * **User Story:** 身為一名投資人，我想要輸入股票代號（如：2330），以便於快速獲取目前的股價、漲跌幅與成交量。
    * **驗收標準 (Acceptance Criteria):**
        1. 支援台灣上市/上櫃股票代號查詢。
        2. 系統需回傳包含：股票名稱、最新成交價、漲跌幅、最高/最低價的 Flex Message 格式卡片。
    * **邊界條件與錯誤處理 (Edge Cases / Error Handling):** 若輸入無效代號，系統需友善回覆「查無此股票代號，請確認後重新輸入」。若為非交易時段，需標示「此為最近一筆收盤資訊」。

* **功能 2: 股票到價提醒 (Price Alerts)**
    * **User Story:** 身為一名忙碌上班族，我想要設定某檔股票的目標價格，以便於在股價觸及時第一時間收到通知。
    * **驗收標準 (Acceptance Criteria):**
        1. 使用者可透過指令（例：`提醒 2330 > 1050`）設定條件。
        2. 系統背景需定期輪詢股價，條件觸發時透過 LINE Push Message 發送通知。
    * **邊界條件與錯誤處理 (Edge Cases / Error Handling):** 限制每位使用者最多設定 5 組提醒以防資源耗盡。到價觸發發送一次通知後即自動解除該設定。

* **功能 3: AI 財經新聞摘要與問答 (結合 LLM)**
    * **User Story:** 身為一名新手投資人，我想要傳送財經新聞連結或提問，以便於讓 AI 幫我摘要重點或解釋專有名詞。
    * **驗收標準 (Acceptance Criteria):**
        1. 使用者傳送新聞網址，Bot 擷取內文並利用 AI 產生 3 點重點摘要。
        2. 使用者直接提問（例：「什麼是殖利率？」），AI 需以白話文解釋。
    * **邊界條件與錯誤處理 (Edge Cases / Error Handling):** LLM 處理較耗時，需先回傳 HTTP 200 給 LINE，並在背景處理，完成後再以 Reply API 或 Push API 回覆。若發生 AI API 額度用盡或逾時，回覆「目前系統繁忙，請稍後再試」。

## 4. 非功能需求 (Non-functional Requirements)
* **效能 (Performance):** Webhook 必須在 1 秒內回傳 200 OK 以避免 LINE Server 重試。股價查詢的回應時間應小於 3 秒。
* **相容性 (Compatibility):** 介面需使用 LINE Flex Message，確保在 iOS / Android 及桌面版 LINE 均能正確顯示。
* **安全性 (Security):** 不得將 LINE Channel Secret 及 API Keys 寫死於程式碼，需使用環境變數。對外部 API 的呼叫需妥善處理憑證。

## 5. UI/UX 與使用者流程 (User Flow)
* **查詢流程:** 使用者開啟聊天室 -> 輸入「2330」-> Bot 解析出數字為股票代號 -> 呼叫外部股價 API -> 組裝 Flex Message -> 回傳給使用者。
* **設定提醒流程:** 使用者點擊 LINE Rich Menu (圖文選單) 的「設定提醒」-> 系統引導輸入格式 -> 使用者輸入「設定 2330 小於 900」-> 系統確認並回覆設定成功 -> 背景排程偵測 -> 觸發時推送訊息。

## 6. 不做的範圍 (Out of Scope)
* 本次里程碑**不包含**真實的下單交易功能（無串接券商 API）。
* **不包含**複雜的技術線圖 (K線圖) 繪製與回測功能。
* **不包含**美股或其他海外股市的報價支援（v1 僅限台股）。
